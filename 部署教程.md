# 复利引擎 上传部署教程（@OxMagic_ 专用版）

> 跟着一步步操作就行，每一步都有截图说明的位置。
> 全程免费，约 30-60 分钟。

---

## 第一步：创建代码仓库（Private 私有）

1. 打开 https://github.com/new
2. 填写：
   - **Repository name**：`compound-engine`
   - **Description**：复利引擎 - BTC 事件合约预测系统
   - **选 Private**（私有仓库，别人看不到代码）✅
   - **不要勾选** Add a README / .gitignore / License（全部留空）
3. 点 **Create repository**

### 上传文件

创建好仓库后，进入仓库页面，点 **uploading an existing file** 链接：

需要上传的文件（全部在 `C:\Users\Lru1\Desktop\复利引擎_迁移包\` 里）：

**根目录文件**（拖拽进去）：
```
compound_engine_server.py    ← 后端主程序
复利引擎.html                 ← 前端页面
requirements.txt             ← Python 依赖清单
Procfile                     ← 启动配置
README.md                    ← 说明文件
auto_trade.py                ← 自动下单模块（可选）
```

**models 文件夹**：
- 点 **choose your files** → 进入 `models` 文件夹 → 全选所有 `.pkl` 文件（22个）→ 上传
- ⚠️ 如果 GitHub 提示文件太大，可以用 Git LFS，但一般不会超过25MB限制

最后底部 **Commit message** 填 `v1.0 初始上传`，点绿色按钮 **Commit changes**。

✅ 代码仓库完成！

---

## 第二步：创建注册码仓库（Public 公开）

注册码必须是公开的，软件才能读取验证。

1. 打开 https://github.com/new
2. 填写：
   - **Repository name**：`compound-engine-license`
   - **Description**：复利引擎注册码验证
   - **选 Public**（公开仓库）✅
   - 同样不要勾选任何初始化选项
3. 点 **Create repository**

### 上传注册码文件

1. 进入仓库，点 **uploading an existing file**
2. 把 `license_repo` 文件夹里的 `licenses.json` 拖进去
3. Commit message 填 `初始注册码`，点 **Commit changes**

4. 点击上传好的 `licenses.json` 文件
5. 点右上角的 **Raw** 按钮
6. 复制浏览器地址栏的 URL，格式类似：
   ```
   https://raw.githubusercontent.com/OxMagic/compound-engine-license/main/licenses.json
   ```
   **把这个 URL 记下来！** 后面配置要用。

✅ 注册码仓库完成！

---

## 第三步：配置注册码 URL（重要！）

软件需要知道去哪里验证注册码，需要把上一步的 URL 配置进去。

1. 打开 https://github.com/OxMagic/compound-engine
2. 找到并点击 `compound_engine_server.py` 文件
3. 点右上角 **铅笔图标** ✏️（Edit this file）
4. 搜索 `LICENSE_URL`（Ctrl+F），找到类似这行：
   ```python
   LICENSE_URL = ""
   ```
5. 改成你上一步复制的地址：
   ```python
   LICENSE_URL = "https://raw.githubusercontent.com/OxMagic/compound-engine-license/main/licenses.json"
   ```
6. 点右上角 **Commit changes** 保存

✅ 配置完成！

---

## 第四步：部署到 Render（让网站上线）

1. 打开 https://render.com
2. 点 **Get Started for Free** → **Continue with GitHub**
3. 授权 GitHub 后，点 **New +** → **Web Service**
4. 找到 **compound-engine** → 点 **Connect**
5. 配置如下：
   ```
   Name:          compound-engine
   Region:        Singapore（新加坡，离国内最近）
   Branch:        main
   Runtime:       Python 3
   Build Command: pip install -r requirements.txt
   Start Command: gunicorn compound_engine_server:app --workers 1 --timeout 120
   Instance Type: Free
   ```
6. 点 **Create Web Service**

⏳ 等待 **5-10 分钟**，第一次部署要安装所有依赖，会比较慢。
在 Logs 页面可以看到进度。

部署完成后你会得到网址，类似：
```
https://compound-engine-xxxx.onrender.com
```

打开它，等 30 秒（冷启动），看到 K 线图就成功了 🎉

## 第五步：防止服务器休眠（推荐）

Render 免费版 15 分钟没人访问会自动休眠，下次打开要等 30-60 秒。

用 UptimeRobot 免费保活：

1. 打开 https://uptimerobot.com → 注册（免费）
2. 点 **Add New Monitor**
3. Monitor Type：选 **HTTP(s)**
4. URL：填你的 Render 网址
5. Monitoring Interval：选 **5 minutes**
6. 点 **Create Monitor**

搞定，服务器会一直在跑了。

## 后续更新代码

需要修改代码时：

1. 去 GitHub 仓库 https://github.com/OxMagic/compound-engine
2. 点要改的文件 → 右上角铅笔 ✏️ → 编辑内容 → **Commit changes**
3. Render 检测到变化会自动重新部署（2-3分钟）

## 常见问题

**Q：部署失败？**
A：Render 控制台点 Logs 看错误，截图发给我。

**Q：网站打不开？**
A：免费版冷启动 30-60 秒，等一下就好。装了 UptimeRobot 就不会了。

**Q：模型加载失败？**
A：检查 models 文件夹的 22 个 pkl 文件是否全部上传了。

**Q：注册码验证失败？**
A：检查 `compound_engine_server.py` 里的 `LICENSE_URL` 是否配对了。
